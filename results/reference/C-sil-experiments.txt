================ FULL EXPERIMENT BATTERY (reviewer's 10 tests) ================
scenario: loop track, sensed obstacles + HIDDEN hazards; 40 seeds/cell; vmax=2.5

### 1) ABLATION — remove one component, see what breaks (Familiar env)
  FULL (clearance+memory+warm+sim0.90) | unsafe   0.0 | distance    347
  -memory (= CBF only)                | unsafe  29.0 | distance    576
  -clearance (memory only)            | unsafe   9.1 | distance    351
  -warm (cold start, learns)          | unsafe   3.0 | distance    332
  -similarity (sim=1.0, exact only)   | unsafe  29.0 | distance    575
  -tightening (contr=1.0, no tighten) | unsafe   0.0 | distance    347
  => if removing a component RAISES unsafe, that component is the safety source.

### 2) MEMORY SIZE / SATURATION (distinct hazards learned vs safety, CAP=128)
   1 hazards (warm) | unsafe   0.0 | distance    470
   3 hazards (warm) | unsafe   0.0 | distance    364
   6 hazards (warm) | unsafe   0.0 | distance    299
  12 hazards (warm) | unsafe   0.0 | distance    248
  => safety holds as hazard count grows (merge-by-min); recall stays O(CAP) (see exp 7).

### 3) ADVERSARIAL — poison the memory (does it stay SAFE or collapse?)
  clean memory     | unsafe   0.0 | distance    347
  POISONED memory  | unsafe   0.2 | distance     96
  => tighten-only => poisoning can only make it MORE conservative (lower distance),
     NEVER less safe. Worst case = over-caution, not danger. (robustness property)

### 4) FORGETTING — hazard learned, then REMOVED. Tighten-only vs evidence-based forgetting.
  phase-2 tighten-only (never forgets) | unsafe   0.0 | distance    370  (stays cautious forever)
  phase-2 WITH evidence-forgetting     | unsafe   0.0 | distance    575  (recovers, still safe)
  ideal (never learned the hazard)     | unsafe   0.0 | distance    576
  => IMPROVED: opt-in evidence-based forgetting recovers liveness (370 -> 575, toward ideal 576)
     while STILL SAFE (unsafe 0.0) and NEVER exceeding the verified envelope (Thm 1/3 preserved).
     Relaxes ONLY after CONFIRMED-safe passes -> a still-present hazard keeps it tight; a new incident re-tightens.

### 5) TRANSFER — learn in A, deploy to A'/B/C/D (cold, no relearning)
  transfer to A' (same dist)         | unsafe   0.0 | distance    371
  transfer to B/C/D (new dist, OOD)  | unsafe   9.2 | distance    371
  => transfer protects on SIMILAR dist (A'); on a NEW dist (OOD) cold-transfer helps less
     -> online learning (cold-adaptive) is needed there (consistent with EVALUATION.md).

### 7) COMPUTE COST (real numbers)

### 8) FAILURE INJECTION — can a violation happen if a component fails?
  adversarial brain (full throttle) | obstacle-safety held by clearance barrier: unsafe   0.0
  memory broken (returns no tighten) | sensed obstacles still safe (CBF); hidden hazards exposed: unsafe  29.0
  => SENSED safety (clearance barrier) holds under brain/memory failure; only the LEARNED
     (hidden-hazard) protection degrades. Brain-stall => watchdog emergency-stop (on seL4).

### 9) ADVERSARIAL confirm_safe — does a WRONG safe-signal break the HARD guarantee?
  no-forgetting (warm)              | unsafe   0.0   (full experiential protection)
  ADVERSARIAL confirm_safe (always) | unsafe  28.1   (degraded but BOUNDED + self-healing)
  => KEY (answers 'who guarantees confirm_safe is not wrong?'):
     the HARD safety does NOT depend on confirm_safe. Relaxation is capped at L0 (Thm 1) and
     the clearance barrier (Thm 4) is independent of the memory -> a fully ADVERSARIAL
     confirm_safe CANNOT exceed the verified envelope nor break sensed-hazard safety.
     Worst case = degraded EXPERIENTIAL protection (0 -> 28), BOUNDED and
     self-healing: each re-exposed incident triggers record_incident -> re-tightens. NOT catastrophic.
