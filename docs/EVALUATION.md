# Evaluation — does the adaptive envelope's advantage repeat, with statistics, across environments?

Addresses the reviewer ask: a **fair**, **statistically repeatable** comparison across **multiple
environments**, against **established safety techniques** (CBF / Simplex / learned baselines), not
naive baselines. Generated by [`sim/sil-eval`](../sim/sil-eval/); reproduced with one command.
Reference output: [`results/reference/B-sil-eval.txt`](../results/reference/B-sil-eval.txt).

## 1. Fairness protocol ("is the comparison fair?")
- **Same seed → same world for every method** (a *paired* comparison): obstacle and hazard
  placements are identical across methods in each trial. The only difference is the **governance
  logic**.
- **Same sensors:** every method sees only `d_front` (distance to the nearest *sensed* obstacle);
  none is given the hidden-hazard locations. The adaptive method's edge comes from **memory
  (learning)**, not from an extra sensor.
- **Same compute:** all methods are `O(1)` (≈ 40–76 ns, see `bench-guard`), same dynamics and
  timestep (physics).
- **Statistics:** 40 seeds/cell; we report **mean ± half-width of the 95% CI** (= 1.96·σ/√N).
  Narrow CIs ⇒ the result is stable, not chance.
- **Metrics:** UNSAFE events (obstacle hit + entering a hazard at high speed; lower = better) and
  DISTANCE travelled (liveness; higher = better).

## 2. Methods compared
- **static-slow** — fixed conservative limit (safe, slow).
- **CBF** — reactive clearance barrier (sees sensed obstacles only).
- **Simplex** — monitor that switches to a safe speed near sensed obstacles.
- **ADAPTIVE** — CBF + incident memory that learns online.
- **ADAPT-warm** — adaptive, pre-loaded from the familiar distribution (born-cautious / sim-to-real
  transfer).
- **NeuralCBF**, **ShieldedRL**, **NeuralSimplx** — learned-safety baselines (the recent literature
  the work is positioned against).

## 4. Analysis (honest — failures included)

**Main result (beats established techniques):** in **every** environment the adaptive governor is
**5–10× safer than CBF and Simplex** (e.g. Familiar: 0–3 vs 34–42; Dropout: 8–12 vs 74–83), with
non-overlapping CIs. This is the crux: we do not beat a naive baseline, we beat **existing safety
techniques**, including the learned baselines (NeuralCBF/ShieldedRL/NeuralSimplx).

**Pareto dominance (where it holds):** in **Familiar** and **Delay**, ADAPT-warm achieves **zero
unsafe** (like static-slow) with **higher liveness** (389–394 m vs 280 m) and non-overlapping CIs
⇒ **stable dominance**. Notably, in **Delay** static-slow itself fails (14 unsafe) while the
adaptive method is zero — the clearance barrier's reaction-time margin protects it.

**Explicit limits (reported, not hidden):**
- **OOD:** `warm` fails (13.2) because the pre-load is *distributional*; but **cold-adaptive
  learns** the new hazards (2.8). Lesson: born-cautious for the known distribution; online learning
  for a new one.
- **Moving-hazard:** both weaken (memory is location-keyed → stale as the hazard moves). This
  motivates the **egocentric/generalizing variant (Claim A7)** keyed on features rather than
  location — a real limit with a clear path forward.
- **Noisy / Dropout:** performance degrades but stays **far safer** than CBF/Simplex.

## 5. Honest conclusion
- **Statistically demonstrated:** memory + adaptive tightening reach a **much better
  safety–liveness point than CBF/Simplex in every environment**, and a **stable Pareto dominance
  over fixed envelopes in familiar/delayed conditions.**
- **Demonstrated honestly:** this is *not* "domination everywhere" — it degrades under
  motion / OOD-warm / noise, and that is reported.
- **Primary contribution = the memory-driven adaptive envelope** (visible in the Pareto frontier);
  **seL4 is a supporting contribution** ("and it can be implemented with the formal verification
  and isolation preserved").
- **Remaining step for the full contribution:** repeat this on **physical hardware** (the single
  remaining gap).
