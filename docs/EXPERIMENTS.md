# Full experiment battery — answering the reviewer's hard questions (12 tests)

Generated by [`sim/sil-experiments`](../sim/sil-experiments/) (40 seeds/cell; same loop scenario:
sensed obstacles + hidden hazards; `vmax = 2.5` unless noted). **Failures are reported honestly —
nothing hidden.** Reference output:
[`results/reference/C-sil-experiments.txt`](../results/reference/C-sil-experiments.txt).

## 1) ABLATION — mechanistic evidence (the most important test)
Remove one component, observe the collapse (Familiar env):

| Variant | unsafe | distance |
|---|---|---|
| **FULL** (clearance + memory + warm + sim 0.90) | **0.0** | 347 |
| **−memory** (= CBF only) | **29.0** | 576 |
| −clearance (memory only) | 9.1 | 351 |
| −warm (cold start, learns) | 3.0 | 332 |
| **−similarity** (sim = 1.0, exact match only) | **29.0** | 575 |
| −tightening (warm pre-load still protects) | 0.0 | 347 |

→ Removing **memory** or **similarity** raises unsafe from 0 to 29. This is **mechanistic
evidence**: the safety source is the **incident memory + similarity generalization**, nothing else.

## 2) MEMORY SIZE / SATURATION (CAP = 128)
| # hazards | unsafe | distance |
|---|---|---|
| 1 | 0.0 | 470 |
| 3 | 0.0 | 364 |
| 6 | 0.0 | 299 |
| 12 | 0.0 | 248 |

→ Safety **holds** as hazards grow (merge-by-min); distance drops (more caution). Recall is O(CAP).

## 3) ADVERSARIAL / TOXICITY — poisoning the memory
| Memory | unsafe | distance |
|---|---|---|
| clean | 0.0 | 347 |
| **poisoned** (60 random/forged incidents) | **0.2** | **96** |

→ Strong robustness property: because tightening is **monotone**, poisoning only makes the system
**more conservative** (distance 96 vs 347), **never less safe**. Worst case = over-caution, not
danger. (Important for cyber-security.)

## 4) FORGETTING — does it stay scared forever? (evidence-based, opt-in)
| Variant (phase 2: hazard removed) | unsafe | distance |
|---|---|---|
| tighten-only (never forgets) | 0.0 | 370 (cautious forever) |
| **+ evidence-based forgetting (opt-in)** | **0.0** | **575 (recovers)** |
| ideal (never learned the hazard) | 0.0 | 576 |

→ Opt-in `confirm_safe`/`with_forgetting` recovers liveness (370 → 575 ≈ ideal 576) **while still
safe (0 unsafe) and never exceeding the verified envelope** (Theorems 1/3 preserved). Relaxes
**only with evidence** — a still-present hazard keeps it tight; a new incident re-tightens
immediately.

## 5) TRANSFER / BORN-CAUTIOUS — learn in A, deploy to A'/B/C/D
| Transfer | unsafe | distance |
|---|---|---|
| A' (same distribution) | **0.0** | 371 |
| B/C/D (new distribution, OOD) | 9.2 | 371 |

→ Transfer protects on a **similar** distribution (A'); on a **new** one it helps less → needs
**online** learning (cold). Consistent with [`EVALUATION.md`](EVALUATION.md).

## 6) STRESS / breaking point — raise speed (unsafe / distance)
| vmax | no-guard | CBF | Simplex | **ADAPTIVE(warm)** |
|---|---|---|---|---|
| 1.5 | 18.0 / 360 | 18.0 / 360 | 25.2 / 506 | **0.0 / 287** |
| 2.5 | 59.0 / 600 | 29.0 / 576 | 33.7 / 670 | **0.0 / 347** |
| 4.0 | 94.4 / 960 | 43.0 / 860 | 41.4 / 827 | **0.0 / 391** |
| 6.0 | 141.6 / 1440 | 58.2 / 1166 | 47.8 / 957 | **0.0 / 394** |
| 9.0 | 212.4 / 2160 | 69.9 / 1400 | 54.0 / 1083 | **0.0 / 394** |

→ The adaptive method **does not collapse**: zero unsafe across all speeds, while CBF/Simplex
degrade. Its breaking point is far beyond the alternatives.

## 7) COMPUTE COST (real numbers)
- memory recall (`effective_limit`, CAP=128, DIM=2): **19 ns/call** (O(CAP), linear)
- memory update (`record_incident`): **472 ns/call** (includes merge scan when full)
- RAM: `sizeof(SafetyMemory<128,2>)` = **2080 bytes (~2 KB)**, static, no heap
- full guard decision: **~40–76 ns** (`bench-guard`)

## 8) FAILURE INJECTION — can a violation happen if a component fails?
| Failure | Result |
|---|---|
| **adversarial brain** (full throttle always) | obstacle-safety held by clearance barrier: **unsafe 0.0** |
| **memory broken** | sensed obstacles still safe (CBF); hidden hazards exposed: unsafe 29.0 |
| **brain stall** | watchdog → emergency stop (verified on seL4, `sel4-guard-pd`) |

→ Sensed-hazard safety (clearance barrier) holds under brain/memory failure; only the **learned**
protection degrades.

## 9) ADVERSARIAL `confirm_safe` — does a WRONG safe-signal break the hard guarantee?
| Variant | unsafe |
|---|---|
| no-forgetting (warm) | 0.0 |
| **adversarial `confirm_safe`** (always) | **28.1** (degraded but **bounded + self-healing**) |

→ KEY (answers "who guarantees `confirm_safe` is not wrong?"): the **hard safety does not depend on
`confirm_safe`.** Relaxation is capped at L0 (Thm 1) and the clearance barrier (Thm 4) is
independent of memory ⇒ a fully adversarial `confirm_safe` **cannot exceed the verified envelope
nor break sensed-hazard safety.** Worst case = degraded *experiential* protection (0 → 28),
bounded and self-healing (each re-exposed incident re-tightens). Not catastrophic.

## 10) SCALABILITY — distinct contexts → entries, recall ns/call, static RAM
| N | CAP | entries | recall | RAM |
|---|---|---|---|---|
| 10 | 16 | 10 | 20.2 ns | 288 B (0.3 KB) |

## 11) FALSE POSITIVES / FALSE NEGATIVES
- **False positives** (ADAPT-warm, Familiar) — tighten-only ⇒ safety held, liveness traded:
  fp 0.000 → 347 m · 0.001 → 87 m · 0.010 → 52 m · 0.050 → 63 m (unsafe stays 0.00 throughout).
- **False negatives** (ADAPTIVE cold) — missed detections leave hazards unlearned:
  fn 0.00 → unsafe 3.0 · 0.25 → 4.0 · 0.50 → 9.0 · 0.90 → 17.0.

## 12) LONG-HORIZON — 200k steps (10×): stable over time?
- first 20k (cold start): unsafe 3 → full 200k: unsafe 3 (≈0.3 per 20k window), distance 3199.
- memory entries after 200k: **3** (CAP=128, bounded by merge-by-min — no growth).

→ Safety is learned in the first laps, then stable; memory bounded; no drift over a 10× horizon.
